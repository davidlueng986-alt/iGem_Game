## 1. 章節承諾

玩家將理解：工程大腸桿菌可以作為生物製造平台，但「細胞產生目標分子」只是開始。重組人類胰島素的實際製造還涉及依製程而異的胰島素鏈或前驅物、回收、折疊與加工、純化、宿主雜質控制、配方、品質檢驗，以及由品質單位依核准程序放行並符合監管要求。病人使用的是經製造和品質控制的藥物產品，不是工程細菌或未處理培養物。

本章使用 1982 年已獲 FDA 核准的重組 DNA 人類胰島素作成熟案例，但所有工廠、批次、數據及角色均為教學虛構。本章不提供醫療建議、劑量、注射方法、實際製程參數或可執行實驗步驟。

## 2. 角色

| 角色 | 聲線 | 戲劇功能 |
|---|---|---|
| 玩家 | 無配音 | 重建生產與品質邏輯，不能自行批准藥物 |
| 何主任，品質負責人 | 精準、寧可停線也不賭運氣 | 代表獨立品質權責與批次放行 |
| 林博士 | 對生物製造興奮，延續第一章成長 | 教導工程細胞功能，但不越過品質權限 |
| 方雅，生物安全主任 | 關注活細胞、宿主雜質、廢物和人員 | 連接生物安全與產品安全，但不把兩者混為一談 |
| 子晴，青年病友代表 | 冷靜、有實際用藥經驗，不作悲情道具 | 提問產品是甚麼、供應中斷與價格由誰承擔 |
| 自動生產系統 | 中性 | 顯示教學流程、批次狀態與偏差，不代替人作放行決定 |

## 3. 不可違反的內容規則

1. 玩家不操作真實發酵、純化或醫療設備。
2. 不顯示菌株、序列、培養條件、純化參數、注射或劑量。
3. 不把「E. coli」寫成一種單一風險；具體菌株和設施另行評估。
4. 工程細胞不會出現在最終藥瓶中；是否符合產品要求由品質系統確認。
5. 品質閘門失敗時必須隔離批次，不能靠玩家話術放行。
6. 本章使用教學批次，不製造病人即將無藥可用的倒數危機。
7. 可及性不是用降低安全標準換取低價；要處理產能、供應、授權和公共支援。

## 8. S01：細胞不是藥瓶

### 玩法

玩家在透明「生產物料圖」中把卡片分成三區：生產平台、製程中的目標物、可放行產品。錯放仍顯示後果。

| Card | Correct Zone | 錯放後果 |
|---|---|---|
| 工程大腸桿菌 | 生產平台 | 放入藥瓶時，何主任立即鎖定並說明不能把活細胞當成產品 |
| 胰島素鏈或前驅物 | 製程中的目標物 | 未加工便放行時顯示結構／功能未證明 |
| 宿主細胞物質 | 待移除／控制 | 放入產品區時品質 gate 失敗 |
| 經加工純化並符合品質要求的胰島素產品 | 可放行產品 | 正確 |

### 對白

| ID | Speaker | Line |
|---|---|---|
| C2-S01-D001 | 子晴 | 有人聽到「細菌製造胰島素」，便以為病人會被注射細菌。你可以把中間發生的事說清楚嗎？ |
| C2-S01-D002P | 林博士 | 工程細胞像工廠，不是最後交到病人手上的產品。工廠做出原料後，還要整理、清除其他東西，再檢查。 |
| C2-S01-D002 | 林博士 | 工程大腸桿菌是生產宿主。依製程，它先產生胰島素鏈或前驅物；其後要回收、正確折疊與加工、純化並控制宿主來源雜質。 |
| C2-S01-D003 | 方雅 | 生物安全關心工程細胞、人員、設施和廢物；產品品質關心最終產品身分、純度、功能和一致性。兩者相連，但不能互相取代。 |

正確分類後設定 `c2_cells_product_separated`。

## 9. S02：從 DNA 到蛋白質

### 場景

玩家進入抽象「細胞工廠」空間。DNA 是固定資料庫，不被畫成程式碼魔法；mRNA 是暫時訊息；核糖體產生多肽。流程不顯示真實序列或操作參數。

### 標準模式節點

```text
設計中的人類胰島素資訊 → DNA 轉錄 → mRNA → 翻譯出鏈／前驅物
→ 回收 → 折疊與加工 → 純化與宿主雜質控制 → 配方與品質檢驗 → 放行決定
```

### 小學模式節點

```text
指令 → 暫時訊息 → 蛋白質前身 → 整理和清除其他物質 → 檢查合格
```

### 關鍵錯誤

| Error | Feedback |
|---|---|
| 把 DNA 直接送入藥瓶 | DNA 資訊不是最終胰島素產品 |
| 跳過折疊／加工 | 有目標成分不代表結構與功能合格 |
| 跳過純化 | 宿主與製程相關物質仍未受控 |
| 把放行放在檢驗前 | 生產部門不能在證據完成前自行批准 |

### 對白

| ID | Speaker | Line |
|---|---|---|
| C2-S02-D001 | 林博士 | 合成生物學讓我們把一項生物功能交給合適宿主，但生物不會像印表機一樣吐出可立即使用的成品。 |
| C2-S02-D002 | 何主任 | 每個箭頭都要回答兩件事：輸出變成了甚麼？下一步靠甚麼證據接受它？ |
| C2-S02-D003 | 子晴 | 對病人而言，「工廠成功表達」不是答案。我們需要的是每一批都符合要求的產品。 |

順序正確後設定 `c2_process_order_valid`。

## 10. S03：四道品質閘門

### 玩法

玩家把八張證據卡分配到四道 gate。小學模式每道只需一張圖像卡；標準模式需同時排除一張看似合理但答錯問題的卡。

| Gate | 正確要回答的問題 | 不足證據例子 |
|---|---|---|
| 身分 Identity | 這是預期的目標產品嗎？ | 只看瓶身標籤 |
| 純度 Purity | 宿主細胞蛋白、殘留核酸、內毒素及製程相關雜質是否受控？ | 只確認有目標成分 |
| 功能 Function／Potency | 結構、折疊／聚集狀態和核准功能測試是否符合要求？ | 只看外觀清澈 |
| 一致與污染控制 | 批次是否在受控流程中保持一致，且無不可接受的微生物／無菌相關或其他污染偏差？ | 只和自己最好的一次比較 |

### 對白

| ID | Speaker | Line |
|---|---|---|
| C2-S03-D001 | 何主任 | 一個檢查不能回答所有問題。身分正確的東西仍可能不純；很純的東西也可能功能不對。 |
| C2-S03-D002P | 子晴 | 名字對、夠乾淨、能工作、每一批都穩定，四件事不能互相代替。 |
| C2-S03-D002 | 子晴 | 我不需要看見所有商業機密，但需要知道品質問題由獨立單位檢查，而不是由想趕進度的人自己批准。 |

四道 gate 完成後設定四個 `c2_quality_*` flags。系統載入 Q-17：身分與功能符合教學要求，宿主來源雜質控制失敗，批次維持隔離。

## 13. S06：新批次

### 修正驗證

玩家把宿主雜質控制節點鎖回流程、加入版本審批 gate，運行新的教學批次 Q-18。系統顯示四道品質 gate 均符合**虛構教學要求**。Q-17 維持原處置決定，不因新批次合格而被追溯放行。

若 `c2_batch_decision = validated_rework`，隔離庫保留 Q-17 黃色容器和「等待預先驗證返工評估」標籤，Q-18 啟動稍晚但材料浪費較少；若 `reject_restart`，Q-17 進入合規報廢／調查路線，Q-18 較快啟動但資源損失較高。兩條路都不改變品質要求。

## 16. 灰盒驗收條件

1. 玩家無法把工程細胞、未加工前驅物或失敗批次放入最終產品區。
2. 玩家能說出至少三個「表達之後」的必要階段。
3. 兩個批次處置選項均有合理取捨，錯誤的掩蓋選項不能前進。
4. 放行由何主任完成；玩家只提交證據。
5. 小學模式不要求理解商業製程名詞，標準模式仍不提供操作參數。
6. 本章不延續第一章污染主題或其感測機制。
