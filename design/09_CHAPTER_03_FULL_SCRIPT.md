## 1. 章節目標

玩家修復一個「輸入移除並超過預期恢復窗後，仍持續產生新 GFP 訊號」的教學基因開關，理解 promoter、operator、repressor、inducer 與 reporter 如何組成可測試的輸入／輸出系統。章節以 LacI／Plac 作經典教學模型，不提供序列、濃度、培養或實驗操作。

玩家必須理解：OFF 不等於絕對零背景；ON 不等於無限輸出；移除輸入後恢復需要時間。修復成功不是讓綠光越亮越好，而是讓輸出符合用途、背景可接受、反應可重複。

## 2. 角色

| 角色 | 功能 |
|---|---|
| 林博士 | 引導基因調控，但會被玩家要求先定義預期行為 |
| 小岑，教學館導覽員 | 代表零基礎觀眾，要求模型可解釋、不是只顯示答案 |
| 方雅 | 界定工程細胞只留在封閉教學設備中 |
| 維修系統 | 顯示元件狀態、背景、延遲與交叉影響 |

## 3. 內容界線

- 本章不延續第一章污染主題，也不使用醫療產品或環境樣本。
- 工程細胞只存在於虛構封閉教學設備。
- `LacI/Plac` 是簡化教學模型；實際表現依宿主、構築、條件和測量而變。
- 本章以已表徵的短壽命 GFP reporter 作教學抽象，讓恢復測試能在關卡時間尺度觀察；一般穩定 GFP 不應被假設會快速消失。
- GFP 是報告訊號，不代表目標功能本身。
- 不把「最亮」當成唯一性能標準。

## 4. 狀態與模式

| Flag | Set |
|---|---|
| `c3_expected_behavior` | 玩家定義 OFF／ON／恢復目標 |
| `c3_fault_repressor` | 找出 Plac/operator 缺少可用 LacI 結合位點 |
| `c3_fault_reporter_leak` | 找出 reporter 持續表達來自錯誤 promoter 路由 |
| `c3_repair_strategy` | `restore_laci` 或 `replace_module` |
| `c3_truth_table_valid` | 無輸入低、輸入高、移除後回低 |
| `c3_failure_reported` | 公開記錄初始失敗與修正 |

| 系統 | 小學模式 | 標準模式 |
|---|---|---|
| 真值表 | 無輸入／有輸入兩格，加一個移除動畫 | 背景、ON 閾值、恢復時間與非目標輸入 |
| 元件 | 開關、守門員、綠色報告員 | Plac/operator、LacI、inducer、GFP |
| 修復 | 兩張方案圖卡 | 比較 leakiness、cross-talk、驗證成本 |

### 3D 關卡配置

細胞城是可步行的巨型電路空間。玩家沿「轉錄軌道」追蹤 RNA polymerase，穿過 Plac 閘門、operator 檢查站、LacI 控制塔和 GFP 報告廣場。故障不是列表答案：玩家要跟著仍在生成的 mRNA 光點走到錯誤路由，再把修復模組放回空間節點。

## 8. S02：進入細胞城

玩家縮入抽象細胞城。Plac 是閘門，operator 是 LacI 辨識位置，LacI 是 repressor，inducer 改變 LacI 的抑制作用，GFP 提供可觀察輸出。

| ID | Speaker | Line |
|---|---|---|
| C3-S02-D001 | 林博士 | 沒有 inducer 時，LacI 結合 operator，降低 Plac 下游轉錄。 |
| C3-S02-D002 | 林博士 | 有 inducer 時，抑制作用下降，GFP 表達提高。這是把自然調控邏輯重新用作教學 reporter 的例子。 |
| C3-S02-D003 | 小岑 | 綠光只告訴我們 reporter 的狀態，不代表細胞做了所有想像中的工作。 |

### 配對互動

| Part | Role |
|---|---|
| Plac/operator | 接收 LacI 調控、控制下游轉錄 |
| LacI | 在無 inducer 時降低輸出 |
| inducer | 改變 LacI 抑制狀態的輸入 |
| GFP | 可由儀器觀察的輸出 |

錯配時先播放因果，不扣生命或時間。

## 9. S03：兩個故障

玩家使用「追蹤轉錄」視圖比較規格與實際路徑。

### 故障一

LacI 模組存在，但 Plac/operator 區的可用 LacI 結合位點在教學模型中缺失，repressor 因而不能降低目標 promoter 的轉錄。

| ID | Choice | Result |
|---|---|---|
| C3-S03-C001 | GFP 太綠，把螢幕調暗。 | 只改顯示，不改系統 |
| C3-S03-C002 | Plac/operator 區缺少可用的 LacI 結合位點。 | 正確；`c3_fault_repressor = true` |

### 故障二

GFP 同時被錯接到 constitutive promoter，因此即使修好 LacI 路徑仍有高背景。

| ID | Choice | Result |
|---|---|---|
| C3-S03-C003 | 刪除 GFP。 | 失去可觀察輸出，不能診斷開關 |
| C3-S03-C004 | 移除錯誤的持續表達路由，保留受 Plac 調控的 reporter。 | 正確；`c3_fault_reporter_leak = true` |

| ID | Speaker | Line |
|---|---|---|
| C3-S03-D001 | 林博士 | 一個現象可以有多個原因。只修第一個看到的問題，背景仍然會高。 |
| C3-S03-D002 | 小岑 | 所以「一次修好」不是英雄時刻；可重現的診斷才是。 |

## 10. S04：兩種修復

| ID | Strategy | Trade-off |
|---|---|---|
| C3-S04-C001 | 恢復 LacI 與 operator 的正確調控，移除錯誤 constitutive 路由。 | 改動較小、開館較快；仍要額外測既有 cross-talk；`c3_repair_strategy = restore_laci` |
| C3-S04-C002 | 換成另一個已表徵的誘導調控模組，再重新驗證整個輸入範圍。 | 模擬 cross-talk 較低；開館延遲、驗證條件加倍；`c3_repair_strategy = replace_module` |
| C3-S04-C003 | 把 GFP 換成更亮 reporter，其他不改。 | 亮度不會修正錯誤邏輯 |

兩個正確方案都可通關。`restore_laci` 在 S05 增加一個非目標輸入測試；`replace_module` 增加第二輪背景與恢復測試，場景時鐘顯示展館延後開放。測試資料、NPC 對白和展板保留選擇。

| ID | Speaker | Line |
|---|---|---|
| C3-S04-D001 | 方雅 | 換模組不等於自動更安全或更好；改動越大，重新確認的範圍也越大。 |
| C3-S04-D002 | 林博士 | 最小修復較快，替換模組可能有長期優點。方案好不好，要回到用途和證據。 |

## 11. S05：三次測試

### 測試順序

1. 無 inducer：背景低於教學 OFF 閾值。
2. 加入抽象 inducer 輸入：GFP 經延遲高於 ON 閾值。
3. 移除輸入：新 reporter 表達先停止，既有短壽命 reporter 不會瞬間消失，經已表徵的系統延遲回到低背景。

標準模式另加入一個非目標輸入；結果不得高於預設干擾界線。所有結果均為教學模擬，不使用真實濃度。

| ID | Speaker | Line |
|---|---|---|
| C3-S05-D001 | 小岑 | 關閉輸入後還亮一會，不一定代表又壞了。已存在的 mRNA 和蛋白質不會瞬間消失。 |
| C3-S05-D002 | 林博士 | 所以規格要寫時間，不只寫 OFF 和 ON。 |

三個狀態符合預期後設定 `c3_truth_table_valid`。

若 `restore_laci`，小岑在結局說：「我們準時開館，也把仍需監測的 cross-talk 寫在展板上。」若 `replace_module`，她說：「我們晚一點開館，換來較低的模擬干擾；新模組的驗證範圍也完整公開。」

## 13. 驗收條件

1. 玩家先定義真值表，才可進入修復。
2. 兩個根本原因都必須找到；不能以顯示亮度修補邏輯。
3. 兩種修復均有持久化取捨，不存在唯一金色答案。
4. OFF 顯示為低背景，不宣稱絕對零。
5. 本章不提供真實 inducer、序列或培養參數。
