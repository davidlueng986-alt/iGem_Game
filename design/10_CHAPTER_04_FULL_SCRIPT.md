## 1. 章節目標

兩個基因開關版本都能運作：版本 A 的最高訊號較高，版本 B 的平均表現較穩定。玩家不能尋找一個脫離情境的「最好」，而要先選擇研究問題，再配置對照與重複，檢查異常值，最後寫出只由資料支持的結論。

本章所有資料均為虛構教學資料。遊戲不教授統計軟體操作或以單一 p-value 取代效果、變異、設計與可重現性。

## 2. 角色

| 角色 | 功能 |
|---|---|
| 玩家 | 決定問題與資料設計，不能在看到結果後偷換主要問題 |
| 小岑 | 主持教學展評審，要求公平比較 |
| 安然，A 組代表 | 重視高輸出，不被寫成造假者 |
| 柏樂，B 組代表 | 重視穩定性，也可能忽略低輸出限制 |
| 何主任 | 把第二章的品質思維帶入資料完整性 |

## 3. 狀態與模式

| Flag | Set |
|---|---|
| `c4_question` | `mean_output` 或 `robustness` |
| `c4_controls_valid` | 程序空白、儀器校準及細胞量／狀態參考配置完成 |
| `c4_replication_valid` | 不以單次最高值代替重複 |
| `c4_prior_repair_loaded` | 第三章修復或 standalone 中性條件已映射到資料條件 |
| `c4_outlier_handled` | 保留、調查並作敏感度比較 |
| `c4_conclusion_valid` | 結論符合預先問題與資料 |
| `c4_data_package_complete` | 原始資料、方法版本、排除理由和限制齊備 |

| 系統 | 小學模式 | 標準模式 |
|---|---|---|
| 數據 | 每個比較單元最多 5 個獨立模擬樣本、圖示平均與散布 | 每個比較單元最多 8 個獨立生物重複、批次／日期標記、技術讀取與一個異常值 |
| 資源配置 | 平均問題用 3 參考＋A/B 各 5 點；穩健問題用分組圖卡顯示 A/B 的 ON/OFF 及兩條件 | 固定 19 槽；依問題分配為高同條件重複或平衡的版本×狀態×條件重複 |
| 異常值 | 「不能因不喜歡便丟掉」 | 技術紀錄、預先規則、含／不含敏感度結果 |

## 5. S00：兩個冠軍

展館中央顯示兩張宣傳圖：A「最高亮度冠軍」、B「最穩定冠軍」。兩張都只展示對自己有利的圖。

| ID | Speaker | Line |
|---|---|---|
| C4-S00-D001 | 安然 | A 有一次非常高，代表它最強。 |
| C4-S00-D002 | 柏樂 | B 每次都差不多，所以它才最好。 |
| C4-S00-D003 | 小岑 | 你們可能都說對了一部分，也可能都把「最好」說得太大。玩家先決定我們真正要回答甚麼。 |

第三章結果在場景中形成可見條件，而不改寫本章 canonical 勝負：`p_c3_repair = restore_laci` 時，第二條件顯示為已知 cross-talk 挑戰並保留監測標籤；`replace_module` 時，第二條件顯示為替換模組後的新版本／日期區組。沒有 summary 時載入中性條件變動，標記 `standalone_default = true`。完成映射後設定 `c4_prior_repair_loaded`。

## 6. S01：先問甚麼

玩家選一個主要問題；另一個成為次要結果，不能在看到資料後互換。

| ID | Question | Trade-off |
|---|---|---|
| C4-S01-C001 | 在指定條件下，哪個版本的平均校正後每細胞 ON 輸出較高？ | 需要足夠同條件重複；`c4_question = mean_output` |
| C4-S01-C002 | 在預期條件變動下，哪個版本維持最一致的 ON／OFF 分離？ | 需要涵蓋多條件；`c4_question = robustness` |
| C4-S01-C003 | 哪一組比較值得贏？ | 不是可測量科學問題 |

| ID | Speaker | Line |
|---|---|---|
| C4-S01-D001 | 何主任 | 問題決定需要甚麼資料。看到結果後才改問題，會令漂亮故事跑在設計前面。 |
| C4-S01-D002 | 安然 | 如果你選穩定性，A 的高值仍可報告，只是不再是主要答案。 |

## 7. S02：十九個槽位

### 基本卡

- 程序空白
- 儀器校準參考
- 經驗證的細胞量／狀態參考
- A 與 B 各自的 ON 或 OFF 獨立生物重複；同一樣本的技術重讀只檢查量測，不增加推論 `n`
- 第二環境條件；用於穩健性問題時會降低每個版本／狀態／條件的重複數

### 有效設計

| Primary Question | 建議配置 | 取捨 |
|---|---|---|
| 平均 ON 輸出 | 三類系統參考＋A-ON／B-ON 在指定條件各 8 個獨立重複，共 19 槽 | 對 OFF 狀態及其他條件推廣較弱 |
| 穩定性 | 三類系統參考＋A/B × ON/OFF ×兩個教學條件各 2 個獨立重複，共 19 槽 | 能直接比較分離與條件變動，但每一格精確度較低 |

玩家可執行任何配置。缺對照或每組只有一個結果時仍產生資料，但報告標記「不能回答主要問題」。修正後設定 `c4_controls_valid` 與 `c4_replication_valid`。

顯示指標是扣除程序背景、通過儀器校準並以經驗證細胞量 proxy 校正的每細胞 reporter 輸出。若 A／B 的細胞量、成長或 viability 狀態有實質差異且不能可靠校正，系統阻止 promoter 排名並要求另行調查。標準模式要求把 A／B 樣本在日期／板位區組中交錯配置，避免版本和位置完全重疊。報告同時顯示效果差異、散布及教學不確定性範圍，不只顯示顯著／不顯著二分。

| ID | Speaker | Line |
|---|---|---|
| C4-S02-D001P | 小岑 | 一個點告訴你「發生過」，幾個點才開始告訴你「通常怎樣」。 |
| C4-S02-D001 | 何主任 | 增加條件廣度和增加同條件重複都合理，但回答的是不同不確定性。資源有限時要留下取捨。 |

## 8. S03：原始資料，不是最佳截圖

系統先顯示所有原始點，再容許切換平均、散布和 ON／OFF 分離圖。不能預設只顯示長條平均。

### Canonical Data Pattern

| Version | Pattern |
|---|---|
| A | 校正後平均 ON 較高，散布較大，其中一點異常高；跨條件 ON／OFF 分離較不一致 |
| B | 校正後平均 ON 較低，散布較小；版本專屬 OFF 與 ON 重複顯示跨條件分離較一致 |

若主要問題是平均輸出，初步結果支持 A 在本次指定條件下平均較高；若主要問題是穩定性，初步結果支持 B 在本次測試條件中較一致。任何情況都不能寫「A／B 在所有用途最好」。

| ID | Speaker | Line |
|---|---|---|
| C4-S03-D001 | 安然 | 原始點看起來比我的宣傳圖亂。 |
| C4-S03-D002 | 小岑 | 亂不是資料的缺點。把變異藏起來，才會讓觀眾誤判。 |
| C4-S03-D003 | 柏樂 | B 比較穩，但若用途真的需要高輸出，它也可能不合適。 |

## 9. S04：那一個奇怪點

A 有一個遠高於其他重複的點。玩家查看對應紀錄，發現讀取器在同一時間出現校準警告，但沒有足夠證據直接宣布數值無效。

| ID | Choice | Result |
|---|---|---|
| C4-S04-C001 | 因為它令 A 看起來最好，所以保留且不說明。 | 不接受 |
| C4-S04-C002 | 因為它很奇怪，所以直接刪除且不留下紀錄。 | 不接受 |
| C4-S04-C003 | 保留原始值、標記校準警告、依預先規則調查，並同時報告含／不含該點的敏感度比較。 | 正確；`c4_outlier_handled` |

| ID | Speaker | Line |
|---|---|---|
| C4-S04-D001 | 何主任 | 異常值可能是錯誤，也可能是我們尚未理解的現象。處理理由必須在看見誰得利之前成立。 |
| C4-S04-D002 | 安然 | 如果拿走那一點，A 的平均仍較高，但優勢沒有宣傳圖那麼誇張。這也要寫。 |

## 10. S05：結論有邊界

### 結論卡

依主要問題載入對應正確卡：

| Question | Valid Conclusion |
|---|---|
| `mean_output` | 在本次指定條件與教學資料中，A 的平均校正後每細胞 ON 輸出較高；變異亦較大，推廣到其他條件需再測試。 |
| `robustness` | 在本次涵蓋的教學條件中，B 的 ON／OFF 分離較一致；它的平均 ON 輸出較低。 |

近似但不足的卡包括：「A 在本條件平均較高，因此先推薦所有場景使用 A」、「B 的 ON 點較集中，因此即使不看版本專屬 OFF 也可宣稱分離更好」、「本批次已涵蓋兩條件，因此可直接推廣到其他宿主」、「校準警告足以把異常值排除而不作敏感度比較」。玩家必須指出每張卡的範圍、對照或排除理由缺口，再選有效卡並附上一項限制，設定 `c4_conclusion_valid`。

| ID | Speaker | Line |
|---|---|---|
| C4-S05-D001 | 小岑 | 好結論不是最響亮，而是讀者知道它能走到哪裡、不能走到哪裡。 |
| C4-S05-D002 | 柏樂 | 今天沒有總冠軍，但我們知道甚麼用途更適合問 A，甚麼用途更適合問 B。 |

## 11. S06：可重現資料包

玩家把以下項目放入公開資料包：

1. 預先選擇的主要問題。
2. 原始資料與圖表，不只放平均。
3. 對照、重複和教學條件描述。
4. 異常值紀錄及含／不含比較。
5. 結論、限制、軟體／內容版本。

漏掉原始資料或排除理由時，另一組無法重現報告，玩家必須補回。完成後設定 `c4_data_package_complete`。

| ID | Speaker | Line |
|---|---|---|
| C4-S06-D001 | 安然 | 我保留了最高點，也保留了它的警告和不含它的結果。這比只放冠軍圖更可信。 |

## 12. 驗收條件

1. 玩家必須在看資料前鎖定主要問題。
2. 不完整對照和單次結果仍能運行，但不能產生有效主要結論。
3. 異常值不可因勝負偏好被靜默加入或刪除。
4. A 或 B 都可能在不同問題下成為較適合方案。
5. 完整資料包是通關條件，不以單一「答對」畫面結束。
