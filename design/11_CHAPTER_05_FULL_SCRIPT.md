## 1. 章節目標

城市循環中心希望處理 PET 廢物。一張過度樂觀的宣傳圖把「實驗室中觀察到酵素作用」直接跳成「把工程塑膠分解菌倒進河口」。玩家要阻止這個推論跳躍，區分 PET 與其他塑膠、酵素與活細胞、低結晶教學材料與真實混合廢物流，並把方案改成受控生產與封閉式回收流程。

本章將酵素 PET 回收描述為具有潛力但受材料結晶度、前處理、製程、能源、成本、下游分離及規模驗證限制的技術。遊戲不宣稱一種酵素能在環境中吃掉所有塑膠，也不批准工程微生物環境釋放。

## 5. S00：一張跳太遠的海報

循環中心海報顯示卡通細菌跳入海中，所有垃圾瞬間消失。方雅貼上「未批准、科學與安全內容錯誤」。

| ID | Speaker | Line |
|---|---|---|
| C5-S00-D001 | 羅工 | 我們的封閉實驗模型看到 PET 材料表面發生酵素作用，宣傳組卻把它畫成下週可以清空河口。 |
| C5-S00-D002 | 海叔 | 河口不是放大的試管。潮水、物種、沉積物和生計都在裡面。 |
| C5-S00-D003 | 方雅 | 先不要聽口號。把「少量試放」放進環境模擬，看看哪些路徑一旦開始便難以追回。 |

| ID | Choice | Result |
|---|---|---|
| C5-S00-C001 | 先少量倒進河裡，看有沒有問題。 | 阻止；小量不是免審理由 |
| C5-S00-C002 | 拒絕釋放，先找能在封閉系統取得證據的替代路徑。 | 正確；`c5_release_rejected` |

若選 C001，模擬顯示潮汐把細胞和 DNA 帶離取樣區，非目標接觸與回收狀態變成未知；玩家無法再把它們完整取回。此後方雅才總結：「小量也可能越過可控制邊界。沒有相稱證據與許可，環境不能成為第一個放大試驗場。」

## 6. S01：它不是所有塑膠

玩家分類材料。PET 水瓶、部分聚酯纖維可進入「可能適用」區，但仍受結晶度、污染和前處理影響；其他塑膠不可因外觀相似便放入。

| ID | Speaker | Line |
|---|---|---|
| C5-S01-D001P | 羅工 | 「塑膠」像「水果」一樣是一大類。能處理一種，不代表全部都能處理。 |
| C5-S01-D001 | 羅工 | PET hydrolase 的證據不能直接推廣到所有聚合物；即使是 PET，不同結晶度和實際廢物流也會改變可處理性。 |
| C5-S01-D002 | 小岑 | 實驗室用的容易處理材料若和真實瓶片不同，海報上的「成功率」便不能直接搬過去。 |

正確分類並選擇「只在已驗證材料範圍陳述」後設定 `c5_claim_scope_valid`。

## 7. S02：從來源到受影響者

玩家把風險節點連成路徑：

| Source | Pathway | Receptor／Impact |
|---|---|---|
| 工程細胞 | 外殼破損、廢水、錯誤處置 | 環境微生物、生態系、人員 |
| 工程 DNA | 細胞或殘留物進入廢物流 | 水平轉移與未知環境命運 |
| 酵素 | 未控制排放、過敏／職業暴露 | 人員、非目標材料或環境 |
| 分解產物與添加物 | 下游分離不足 | 水體、再製品質、廢物處理 |
| 前處理與能源 | 高資源需求 | 碳排、成本、環境負擔 |

| ID | Speaker | Line |
|---|---|---|
| C5-S02-D001 | 方雅 | 換成純化酵素通常能移除活細胞釋放問題，但不會令其他風險消失。 |
| C5-S02-D002 | 海叔 | 我關心的不只是「有沒有細菌」，也包括廢水去哪裡、誰長期監測、若漁獲受影響誰停機。 |

全路徑完成後設定 `c5_pathways_mapped`。

## 8. S03：三條路

| Strategy | Result |
|---|---|
| 直接釋放工程微生物到河口 | 不接受；不進入後續比較 |
| 工程細胞在受控設施生產酵素，去除／控制細胞後由封閉反應器使用酵素 | `c5_contained_strategy = enzyme_only`；降低活細胞環境暴露，增加酵素生產與分離成本 |
| 工程細胞只在封閉工業反應器工作，所有細胞、DNA、流出物和產物受控 | `c5_contained_strategy = closed_whole_cell`；流程較少轉換，活細胞封閉與廢物要求較高 |

後兩個方案都可進入概念評估，不宣稱已可工業化。

| ID | Speaker | Line |
|---|---|---|
| C5-S03-D001 | 羅工 | 純化酵素方案減少活細胞風險，但酵素不是免費出現；它仍要生產、品質控制和回收。 |
| C5-S03-D002 | 方雅 | 封閉 whole-cell 方案不等於釋放。真正問題是邊界、失效、流出物、監測和停止規則能否被驗證。 |

## 9. S04：證據階梯

玩家排列：

1. 在標準化 PET 材料上確認酵素與對照差異。
2. 比較材料結晶度、污染和非目標材料，界定適用範圍。
3. 使用代表性 post-consumer feed 在封閉小型系統測試。
4. 驗證物料衡算、產物品質、流出物與廢物控制。
5. 確認可回收單體的品質、分離與再聚合是否真正形成閉環，而不只觀察表面水解。
6. 與減量、重用、機械回收及其他處理路線比較能耗、前處理、水、成本與完整生命週期影響。
7. 進入獨立安全、環境、法規及持份者審查。
8. 只有通過 gate 才考慮受控 pilot；環境釋放不是預設終點。

| ID | Speaker | Line |
|---|---|---|
| C5-S04-D001 | 小岑 | 每上一級都會加入更接近真實世界的新問題。跳級不會令問題消失，只會令我們看不見。 |
| C5-S04-D002 | 羅工 | 「可分解」不等於經濟、低碳或能處理混合垃圾。技術和整個流程要一起評估。 |

設定 `c5_evidence_ladder_valid`。

## 11. S06：不離開實驗室也能前進

玩家以四張卡完成公開說明：

1. 對特定 PET 材料有潛力，不是所有塑膠消失器。
2. 實驗室資料支持下一級封閉驗證，不支持環境釋放。
3. 選定方案如何控制細胞／酵素、流出物、廢物和人員暴露。
4. 成本、能耗、材料適用、長期監測和停止條件仍待驗證。

近似但不足的卡包括「enzyme-only 已移除活細胞，所以可省略人員暴露與流出物 gate」、「小型封閉 pilot 可先做，再補代表性廢物流證據」、「觀察到表面水解便可把單體回收和閉環品質列為完成」。玩家指出缺少的路徑或證據後，完成設定 `c5_public_statement_valid`。

若 `c5_contained_strategy = enzyme_only`，公開剖面顯示工程細胞留在生產區、酵素進入封閉回收反應器，代價欄顯示額外分離和品質成本。若 `closed_whole_cell`，畫面顯示雙重封閉 reactor、完整細胞／DNA／流出物回收，代價欄顯示更高封閉與廢物負擔。

若 `c5_lifecycle_choice = local_pilot`，海叔取得現場停止與監測席位；若 `shared_facility`，他取得跨區事故通知、運送追蹤和本地資料回傳權。選擇由世界場景和結局保留，不只存為旗標。

| ID | Speaker | Line |
|---|---|---|
| C5-S06-D001 | 羅工 | 我們沒有得到一個可以倒進海裡的產品。我們得到一個更清楚、更安全的下一步。 |
| C5-S06-D002 | 海叔 | 不釋放不代表反對技術。它代表河口不是用來補做實驗室缺少的證據。 |
| C5-S06-D003 | 方雅 | 負責任設計有時不是加一道防護，而是改變整個使用方式。 |
