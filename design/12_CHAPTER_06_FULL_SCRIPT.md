## 1. 章節目標

工程酵母可生產青蒿酸等前驅物，後續仍需化學轉換、製成核准青蒿素衍生物、納入含夥伴藥物的 artemisinin-based combination therapy（ACT）產品，以及完成藥品製造品質流程。本章不把工程酵母、前驅物或青蒿素單方寫成可直接交付病人的完整治療，也不提供任何製造或用藥步驟。

玩家要理解：合成生物學可以增加供應來源和穩定性，但技術成功不會自動帶來低價、公平分配或平順轉型。植物來源、農民、製造者、公共採購和病人社群都是系統的一部分。

## 2. 角色

| 角色 | 功能 |
|---|---|
| 阿米娜，公共健康採購員 | 關注供應穩定、品質與可負擔性 |
| 美蘭，黃花蒿種植合作社代表 | 關注價格暴跌、轉型時間和生計 |
| 羅博士，工程酵母團隊 | 解釋前驅物生產與技術限制 |
| 卡里姆，病人社群代表 | 要求供應決策以取得和治療系統為中心，不被當作數字 |
| 玩家 | 建立供應方案，不能決定臨床治療或個人用藥 |

## 3. 內容界線

- 不提供抗瘧疾用藥建議、劑量或療程。
- 清楚寫明工程酵母生產的是前驅物，後續還有轉換、製造和品質程序。
- 不宣稱半合成來源完全取代植物來源或必然更便宜。
- 不把農民描寫成阻礙技術；也不把新技術描寫成背叛傳統。
- 所有價格和供應資料均為教學模型。

### 3D 關卡配置

供應室是一張可步行的世界網路：玩家搬運供應 token 經過農場、酵母前驅物工廠、轉換／製造、品質、公共採購、區域倉庫和偏遠診所。氣候與停機事件會實際關閉路段；玩家要在地圖上重配來源和 buffer，而不是只看試算表。

## 6. S00：被刪掉的人

供應模型為求簡化，只剩「成本」和「產量」。美蘭的合作社、卡里姆的地區診所和品質節點全部被隱藏。

| ID | Speaker | Line |
|---|---|---|
| C6-S00-D001 | 美蘭 | 模型把植物價格寫進去了，卻沒有寫價格突然跌下來時，農民下一季怎樣生活。 |
| C6-S00-D002 | 卡里姆 | 它也沒有寫產品到了國家，是否真的到達偏遠診所。 |
| C6-S00-D003 | 阿米娜 | 先把被簡化掉的人和關卡放回來，否則最低成本只是對某些人最低。 |

玩家找回農民、前驅物轉換、品質放行、採購、地區配送、病人取得節點，設定 `c6_missing_people_found`。

## 7. S01：兩條來源，不是兩種藥

### 植物來源概念鏈

```text
黃花蒿種植與收成 → 提取與製成核准衍生物／ACT 產品 → 品質單位放行並符合監管要求 → 採購配送
```

### 半合成來源概念鏈

```text
工程酵母發酵產生青蒿酸前驅物 → 後續化學轉換 → 製成核准衍生物／ACT 產品 → 品質單位放行並符合監管要求 → 採購配送
```

| ID | Speaker | Line |
|---|---|---|
| C6-S01-D001 | 羅博士 | 工程酵母增加的是前驅物來源，不是把完整 ACT 治療系統塞進發酵槽。 |
| C6-S01-D002 | 阿米娜 | 最終產品仍要符合相同的品質和監管要求。來源不同，不代表可以分成「真藥」與「假藥」。 |
| C6-S01-D003 | 美蘭 | 植物路也不是落後按鈕。它有知識、土地、生計和自己的供應風險。 |

順序正確設定 `c6_chain_valid`。

## 8. S02：三場衝擊

玩家把兩條來源放入三個教學事件：

| Event | 只靠植物 | 只靠單一半合成工廠 | 多來源 |
|---|---|---|---|
| 極端天氣影響收成 | 高風險 | 影響較小 | 緩衝 |
| 主要工廠停機 | 影響較小 | 高風險 | 緩衝 |
| 需求突然上升 | 受生長週期限制 | 受產能與轉換限制 | 仍需 buffer 和採購協調 |

| ID | Speaker | Line |
|---|---|---|
| C6-S02-D001 | 卡里姆 | 多一條來源可以降低某些風險，但兩條都可能有單點故障。 |
| C6-S02-D002 | 羅博士 | 「生物製造可擴展」不等於產能可以瞬間出現。設施、原料、轉換和品質都需要時間。 |

玩家建立至少一項安全庫存／緩衝和兩個來源，設定 `c6_shock_response_valid`。

## 9. S03：價格不是唯一答案

玩家選擇儀表板指標。只有「每單位最低價格」不能通關；必須加入：

- 缺貨日數
- 品質合格率
- 偏遠地區覆蓋
- 供應恢復時間
- 農民收入波動／轉型時間
- 採購透明度

| ID | Speaker | Line |
|---|---|---|
| C6-S03-D001 | 阿米娜 | 價格很重要，但沒有貨的低價只是空格。 |
| C6-S03-D002 | 美蘭 | 平均價格穩定，也可能把波動全部推給農民。要看誰承擔風險。 |

設定 `c6_access_metrics`。

## 10. S04：兩個可行方案

| ID | Strategy | Trade-off |
|---|---|---|
| C6-S04-C001 | 雙來源採購＋公共 buffer；預先定義何時增加半合成或植物採購。 | 韌性高、庫存及協調成本高；`c6_strategy = dual_source_buffer` |
| C6-S04-C002 | 區域授權與製造合作＋植物來源保留；共享品質能力與採購資料。 | 地區能力可提升、技術轉移和治理較複雜；`c6_strategy = regional_partnership` |
| C6-S04-C003 | 立即只選當下最便宜來源。 | 不接受；忽略波動、單點故障與轉型成本 |

兩個有效方案都要通過三場衝擊，並顯示不同指標結果。

## 11. S05：轉型要有時間

玩家先模擬四張轉型卡的影響，再把結果交給美蘭和合作社代表決定；玩家不能代替受影響群體選擇。

| Package ID | Card | Meaning |
|---|---|---|
| `notice_transition_purchase` | 最低通知期＋過渡採購／收入緩衝 | 農民有時間調整，不把轉型成本全推給一方 |
| `notice_new_roles` | 最低通知期＋替代作物或新供應角色支援 | 由合作社決定新路線，不強迫單一選項 |
| 不序列化 | 不通知，讓市場自行淘汰 | 顯示收入急跌與下一季供應反彈，不列為可接受方案 |

系統可依前面策略提出建議：`dual_source_buffer` 預設提示 `notice_transition_purchase`，`regional_partnership` 預設提示 `notice_new_roles`。合作社代表可接受、要求改用另一個有效 package，或拒絕條件並要求玩家修訂；只有合作社確認後才把精確 package ID 寫入 `c6_transition_plan`，並在結局地圖保留其選擇。

| ID | Speaker | Line |
|---|---|---|
| C6-S05-D001 | 美蘭 | 我不要求技術停下來。我要求轉型不要假裝今天的決定不會改變明年的生活。 |
| C6-S05-D002 | 卡里姆 | 也不要讓保護生計變成維持缺貨的理由。兩種責任要一起設計。 |

合作社確認的有效 package 設定 `c6_transition_plan`；未確認或「不通知」不通過本場景 gate。

## 12. S06：成果如何到達

玩家完成四卡說明：

1. 工程酵母生產前驅物，後續仍需轉換、製造和品質放行。
2. 半合成來源可補充供應，但不保證自動低價或完全取代植物。
3. 選定方案如何面對氣候、工廠停機和需求上升。
4. 農民轉型、地區配送、品質與病人取得如何被量度。

| ID | Speaker | Line |
|---|---|---|
| C6-S06-D001 | 羅博士 | 技術的工作是增加可能性；採購、授權和公共政策決定可能性怎樣被分配。 |
| C6-S06-D002 | 美蘭 | 我的合作社沒有被寫成阻力，也沒有從模型裡消失。這才算共同設計。 |
| C6-S06-D003 | 卡里姆 | 最終指標不是發酵槽做了多少，而是品質合格的產品能否穩定到達需要的人。 |

若 `dual_source_buffer`，地圖顯示兩條來源和公共庫存吸收兩次衝擊，但維持庫存成本較高；若 `regional_partnership`，地圖顯示更多區域製造與品質節點，初期建置較慢但偏遠覆蓋改善。美蘭所選轉型方案在農場區形成可見的新採購或培訓站。

設定 `c6_statement_valid`。

## 13. 參考與驗收

- Paddon & Keasling, [Semi-synthetic artemisinin](https://doi.org/10.1038/nrmicro3240)，作歷史技術案例；正式發布前須另以當期來源查核半合成與植物來源的實際市場角色，不能把 2014 年資料當成 2026 年供應現況。
- 玩家必須保留至少兩個供應來源或實質備援，不得以即時最低價格單獨通關。
- 腳本不得把工程酵母前驅物描述成可直接使用的藥物。
- 農民與病人代表都要改變方案，不只提供情緒對白。
