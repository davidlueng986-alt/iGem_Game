## 1. 章節目標

玩家管理一個虛構的合成生物學設計協作平台。相同工具可支援教育、環境及醫療，也可能被誤用。章節不展示病原體、毒素、危險功能、序列、製造方法或提升危害的操作細節；所有高風險內容以抽象能力卡表示。

目標不是把所有合成生物學保密，也不是把開放等同不負責任。玩家要根據具體能力、申請者、用途、材料、規模、資訊細節和控制，決定公開、受控合作、要求更多資料、拒絕或升級審查。

## 3. 內容界線

- 高風險卡只顯示「可能提升高後果生物能力」，不顯示對象、序列或方法。
- 不使用「壞人長相」判斷；看證據、能力、存取和行為。
- 不把特定國籍、年齡、職業或政治立場當成風險代理。
- 拒絕必須有理由、紀錄、覆核和申訴途徑。
- 低風險教育資訊應盡量開放，不以安全為名不必要封鎖。

## 7. S01：風險不是一個標籤

玩家組合六個維度：

1. 能力：內容實際能做甚麼。
2. 使用者：身分、資格、過往行為是否可驗證。
3. 用途：目的、受影響者和合法性是否清楚。
4. 材料與環境：需要甚麼資源，在哪裡使用。
5. 規模與可逆性：影響範圍、能否停止或回復。
6. 資訊細節：概念摘要、分析資料或可直接執行內容。

錯誤維度包括國籍、外表、是否同意研究員等。設定 `c7_risk_dimensions_valid`。

| ID | Speaker | Line |
|---|---|---|
| C7-S01-D001 | 方雅 | 善良目的不能代替控制；陌生身分也不能自動證明惡意。決定要能指出具體證據。 |
| C7-S01-D002 | 阿哲 | 這樣公眾才能問「為甚麼限制」，而不是只聽到「相信秘密委員會」。 |

## 9. S03：受控合作

ENV-12 有合理用途和已驗證合作機構。審核發現科學方法與非敏感結果可以公開，但原始包混入具名合作方營運資料、設施位置／安全細節及契約限制。限制原因不是「資料很詳細」本身，而是已辨識的具體風險。玩家設計：

- 公開科學方法、非敏感原始／聚合結果和限制。
- 經審核成員取得最小必要資料。
- 工作區記錄存取、到期、下載和版本。
- 分階段提供資料，達到 gate 才增加權限。
- 對營運資料作去識別／刪除設施安全細節；風險解除後恢復公開。

| ID | Choice | Trade-off |
|---|---|---|
| C7-S03-C001 | 一次提供全部資料，之後再看用途。 | 不接受；失去最小權限 |
| C7-S03-C002 | 科學內容盡量公開；具名營運／設施資料在去識別前採分階段最小權限。 | 正確；`c7_case_environment = controlled_collaboration`；`c7_access_controls_valid = true` |
| C7-S03-C003 | 永久拒絕所有跨機構合作。 | 不相稱；阻止可管理的公共利益 |

| ID | Speaker | Line |
|---|---|---|
| C7-S03-D001 | 羅工 | 科學方法沒有因合作而變秘密。真正受控的是尚未去識別的營運與設施資料，完成處理後應重新評估開放。 |

## 10. S04：資料不足不是批准

UNV-99 無法驗證身分、用途和控制，卻要求高影響完整設計包。

| ID | Choice | Result |

| C7-S04-C002 | 因身份和用途無法驗證而永久拒絕，不提供覆核或補件。 | 不接受；暫緩有依據，永久裁決仍需相稱程序 |
| C7-S04-C003 | 暫緩、要求必要資料並升級獨立責任委員會；保存理由和申訴途徑。 | 正確；`c7_case_unverified = hold_escalate` |

| ID | Speaker | Line |
|---|---|---|
| C7-S04-D001 | 方雅 | 「未證明安全」不是「已證明惡意」，但在高後果能力前足以暫緩並升級。 |
| C7-S04-D002 | 阿哲 | 公開報道可以說明程序和原則，不必公布申請細節或把未知寫成罪名。 |

## 11. S05：異常下載事件

ENV-12 的一個已批准帳號在短時間要求超出原範圍的大量資料。這可能是帳號被盜、腳本錯誤、研究需求改變或濫用，不能只猜一種。

玩家排列事件流程：

1. 暫停超範圍下載，不刪除紀錄。
2. 保留 audit log 和相關證據，最小化不必要個資暴露。
3. 通知帳號持有人、安全負責人及必要委員會。
4. 驗證身分、原因、影響和是否有資料外流。
5. 依結果恢復、限制、撤銷或升級，保留申訴與事後檢討。
6. 通知受影響合作方，修正控制而非只處罰個人。

近似但不足的路徑包括：「只暫停帳號但不保存超範圍要求紀錄」、「先通知所有平台使用者並附完整事件日誌以示透明」、「帳號曾獲批准，所以只寄提醒而不限制新下載」。玩家指出證據保存、資訊最小化或即時控制缺口後，完成設定 `c7_incident_response_valid`。

| ID | Speaker | Line |
|---|---|---|
| C7-S05-D001 | 方雅 | 過去批准不是永久空白支票；異常也不是自動定罪。先停止可能擴大的行為，再保存和查證。 |
| C7-S05-D002 | 羅工 | 事件令合作慢了一點，但清楚程序比私下猜疑更能維持長期合作。 |

## 12. S06：說明邊界

玩家組合公共摘要：

1. 低風險教育資料公開，附來源、安全與限制。
2. 可管理的較詳細合作使用分階段、最小權限、到期和 audit。
3. 高後果且資料不足的申請暫緩並交獨立覆核，不公開敏感細節。
4. 異常事件按程序調查，決定與申訴留有紀錄。

近似但不足的卡包括：「公開受控資料類別但省略申訴途徑」、「以『機密』概括所有限制而不說明原則與覆核」、「公開完整事件日誌來證明平台透明」。玩家修正程序問責或資訊危害缺口後，完成設定 `c7_public_summary_valid`。

| ID | Speaker | Line |
|---|---|---|
| C7-S06-D001 | 梁老師 | 我的教材仍然開放。安全沒有變成把所有教育鎖起來的藉口。 |
| C7-S06-D002 | 羅工 | 合作仍然可重現。開放也沒有變成一次交出全部能力的藉口。 |
| C7-S06-D003 | 阿哲 | 公眾知道原則、程序和問責；真正可能造成資訊危害的細節沒有被當成宣傳素材。 |
| C7-S06-D004 | 方雅 | 合成生物學不是生物武器的同義詞。正因為它能做很多有價值的事，我們才需要按具體能力負責任地開放。 |

世界結果：公開資料館新增 EDU-01 完整教材；ENV-12 的科學資料進入公開館，具體營運資料留在有到期日的工作區；UNV-99 在覆核室顯示「等待資料／可申訴」，不被公開貼上犯罪標籤。三個結果同時可見。章節完成時序列化 `p_c7_access = { education: open, environment: controlled_collaboration, unverified: hold_escalate }`。

## 13. 驗收條件

1. 低風險教材必須可公開，不允許全封鎖作唯一安全答案。
2. 高風險案例不得被公開或由玩家單獨最終裁決。
3. 不顯示任何可執行危險能力、序列、材料或方法。
4. 異常事件同時保護安全、證據、程序正義和申訴。
5. 公開摘要需解釋原則，不洩漏敏感申請內容。
