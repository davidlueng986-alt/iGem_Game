## 1. 終章承諾

澄灣食物銀行不是向研究站索取一個指定技術，而是提出問題：部分捐贈食物在運送中可能經歷不當溫度，現有電子記錄器太少，志工也不能靠猜測決定食物安全。玩家要和食物銀行、食物安全人員、志工、捐贈者及使用者共同定義一個輔助提示工具。

候選概念是**封閉、與食物隔離、不以活細胞作功能載體的 cell-free 教學標籤**。它必須和增加電子記錄器、改善流程及合格既有 time-temperature indicator 等基準方案比較；只有提供經驗證的額外價值才進入 pilot。cell-free 候選以抽象時間整合、溫度感應和鎖存輸出形成時序邏輯，達到條件後提示不能因重新降溫而消失。這是虛構教學設計，不是已驗證食物安全產品，也不取代正式冷鏈記錄、食物安全規則或專業判斷。

## 4. 內容與安全界線

1. 不顯示 cell-free 系統的序列、配方、製備或可執行步驟。
2. 標籤不接觸食物，位於外包裝指定位置並有完整物理隔離。
3. cell-free 表示功能不依賴完整活細胞，不保證成品天然沒有可存活污染；若規格宣稱無可存活細胞，必須由放行測試確認，並評估宿主成分、內毒素／過敏相關風險、DNA/RNA、蛋白質、染料、包材、洩漏和廢物。
4. 標籤只提供輔助提示，不取代正式溫度記錄、食品分類、保質資訊或主管規則。
5. 不以真實食品安全溫度或時間數字作遊戲答案；閾值由虛構 Food Safety Profile 提供。
6. Pilot 使用模擬包裝及非供食用教學物料；正式現場使用需另行審批。

### 問題定義卡

玩家選擇：

- **使用者：** 受訓志工和食物安全人員。
- **問題：** 部分箱子的冷鏈記錄不足，需要輔助分流提示。
- **非目標：** 不診斷食物安全、不接觸食物、不取代正式規則。
- **成功：** 易讀、低成本、低誤報、可追蹤、能清楚觸發「隔離並確認」。

近似但不足的卡包括：「標籤未觸發且電子參考正常便直接返回一般流程，不需食安人員確認」、「只改善掃描速度與畫面而不量度冷鏈分流缺口」、「顏色可見且單價低便算成功，誤報與追蹤留到 pilot 再定」。玩家先觀看流程後果，再修正並設定 `f_problem_statement_valid`。

## 10. S03：兩把鑰匙

### 邏輯需求

候選標籤使用三個抽象狀態：

- `T = 1`：當下溫度條件超過 Food Safety Profile 的教學閾值。
- `D = 1`：只在 `T` 持續存在時，時間整合器達到完成狀態。
- `L = 1`：輸出已鎖存。狀態規則為 `L_next = L_current OR (T AND D)`。

本關卡的虛構 Food Safety Profile 明確採「連續暴露」教學規則：在 `D` 完成前若 `T` 回到 0，進度重設；再次偏高要重新計時。若其他食品 profile 採累積或溫度加權模型，必須載入另一套專業規則，不能沿用本關卡二元答案。

| 狀態 | T | D | L before | L after／Output |
|---|---:|---:|---:|---|
| 正常冷藏 | 0 | 0 | 0 | 0／OFF |
| 短暫偏高 | 1 | 0 | 0 | 0／OFF |
| 偏高持續足夠時間 | 1 | 1 | 0 | 1／「隔離並確認」 |
| 觸發後重新降溫 | 0 | 0 | 1 | 1／保持提示 |
| 強制單一 D 測試 | 0 | 1 | 0 | 0／AND 不應誤觸發 |
| 未完成前降溫後再次偏高 | 1 | 0 | 0 | 0／依本 profile 重新計算連續時間 |
| 閾值附近反覆變動 | 0↔1 | 0 | 0 | 0／由 profile 判定 T；未持續足夠時間不觸發 |

玩家組合溫度感應模組、時間整合模組、AND 邏輯、鎖存輸出模組與 reporter。介面只顯示功能方塊，不顯示分子實作。

| ID | Speaker | Line |
|---|---|---|
| F-S03-D001P | 林博士 | 熱度和時間兩把鑰匙都轉動，提示會鎖住；之後降溫也不能把歷史擦掉。 |
| F-S03-D001 | 林博士 | AND 條件觸發鎖存輸出，避免短暫單一輸入直接提示，也避免降溫後抹除歷史；背景、時間整合誤差和 reporter 延遲仍需量度。 |
| F-S03-D002 | 黎主任 | 即使 ON，文字也只能是「隔離並確認」，不能寫「危險」或「可安全食用」。 |

這只是教學近似。真實食物風險依食品類別、初始狀態及非線性時間－溫度歷程而變，不能由通用二元 gate 取代專業模型。

七個狀態轉移和文案正確設定 `f_latched_state_valid`。

## 12. S05：數據要求重做一次

若有 `p_c4_question`，結果介面依 `mean_output` 或 `robustness` 預設開啟平均／散布或跨情境分離圖，但玩家可隨時切換，原始點始終可見；沒有 summary 時使用中性全點視圖。

### 第一輪結果

- 高濕情境出現背景升高。
- 一張標籤的顏色相機可辨、人眼在低光下難分。
- 一個 T=1,D=0 樣本接近閾值，但重複不一致。

### 玩家處理

第二輪與預先配置的 challenge cases 顯示：高濕背景改善；色覺替代可由形狀／文字判讀；低光或延遲掃描保留讀取時間並進入「不確定／需確認」；包材微裂使該標籤失效並隔離，不以電路訊號放行；已知 ON 的漏報觸發批次停止與偽陰性調查；已知 OFF 的誤報觸發不必要隔離／丟棄影響檢討。邊界樣本不硬塞入 OFF 或 ON。所有情境均有結果與下一步後設定 `f_edge_cases_valid`。

## 15. S08：Pilot Gate

### 架構決定與 Pilot／No-pilot 設計

S08 先並排顯示基準與候選在預先鎖定指標上的結果。只有 `f_comparison_plan_valid`、`f_latched_state_valid`、`f_controls_valid`、`f_edge_cases_valid`、`f_quality_release_valid`、`f_open_package_valid` 及取得方案均完成，才可作最後決定：

- 基準流程已滿足不可退讓需求，或候選沒有證明額外價值／仍有未受控 gate：設定 `f_solution_architecture = workflow_baseline`。
- 候選在覆蓋、可讀性或可追蹤性至少一項顯示與用途相關的額外價值，且沒有違反持份者條件及安全／品質 gate：可設定 `f_solution_architecture = cell_free_hybrid`，進入受限比較 pilot。
- 即使候選通過，玩家仍可因成本、維護或廢物負擔選擇基準，但必須在公開說明記錄理由；不能以「新技術較有趣」作決定。

若有 `p_c5_pilot`，`local_pilot` 載入社區現場停止與監測席位提醒，`shared_facility` 載入跨區事故通知、運送追蹤及本地資料回傳提醒；沒有 summary 時要求玩家自行選擇相稱治理檢查。

- 只用模擬外包裝與非供食用教學物料。
- 電子參考記錄器與標籤並行。
- 預先定義偽陰性／偽陽性、破損、無法讀取及志工偏差的停止門檻。
- 食物安全人員擁有暫停與下一階段建議權。
- 由不屬於設計團隊的食物安全／品質審查者獨立評估資料與下一階段建議。
- 志工和社區代表可回報、申訴和查看結果。
- 未達 gate 時回到設計，不以「已投入很多」作繼續理由。

若 `f_solution_architecture = cell_free_hybrid`，上述要求組成受限模擬 pilot。若為 `workflow_baseline`，則設定「不啟動 cell-free pilot，擴充基準方案、保存比較與失敗資料、指定重新評估 gate」的負責任 no-pilot 計畫。兩者均經獨立審查後設定 `f_pilot_plan_valid`。

### 最終公開說明

玩家組合五張卡：前四張固定載入，第五張只載入與 `f_solution_architecture` 相符的 5A 或 5B，不同架構的聲明不會同時出現在可選池。

1. 問題由食物銀行提出，工具只輔助冷鏈分流。
2. cell-free 候選以 AND 條件觸發鎖存提示，重新降溫不會抹除已記錄歷史；基準方案仍作比較。
3. 第一輪高濕、低光和邊界問題如何改變第二版。
4. 標籤不接觸食物，也不以活細胞作功能載體；是否無可存活細胞仍需放行確認，且保留材料、批次、廢物和誤用風險。
5A. `cell_free_hybrid`：目前只批准比較用模擬 pilot；正式使用仍需證據與主管審查。
5B. `workflow_baseline`：目前不啟動 cell-free pilot；先擴充基準方案並保留候選研究與重新評估 gate。

近似但不足的卡包括：「標籤通過本輪模擬，所以可作食物安全判定」、「cell-free 候選不使用活細胞，因此只需保留包材風險」、「本食物銀行同意後可直接推廣到其他站點」。玩家修正用途、風險或授權範圍後，完成設定 `f_final_statement_valid`。

## 17. 驗收條件

1. 玩家必須完成需求、候選電路、測試、風險、品質、取得、資料及 pilot／no-pilot 八類輸出。
2. 標籤不得被描述為食物安全證明或正式產品。
3. cell-free 不得被描述成零風險；包材和廢物必須入系統。
4. 小學模式目標 25 分鐘、上限 30 分鐘，不需要閱讀標準模式的長段落。
5. 前七章至少各有一項概念在玩法中被再次使用。
6. 結局不以高分或所有 NPC 無條件支持判定成功。

## 18. 參考與核准

- Cell-free 時間／溫度標籤為虛構教學概念，正式版本必須由 cell-free、食品安全、包材、可及性及法規專家審核。
- 不可把任何真實食品安全閾值硬編入遊戲而沒有地區、食品類別與主管來源。
